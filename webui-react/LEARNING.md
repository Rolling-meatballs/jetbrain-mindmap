## 阶段 1 · Tailwind:用「改样式」学会它（你说想上手,这就是上手）

> Tailwind 的全部秘密:样式不写在单独的 CSS 文件里,而是一个个小「class」直接挂在元素的 `className="..."` 上。改 class = 改样式。

**1-1 🙋 改一个按钮的颜色**
- 做:在 `Toolbar.tsx` 找到 Save 按钮的 `className`,把 `bg-slate-800` 改成 `bg-blue-600`,存盘。
- 会看到:Save 按钮从深灰变蓝。
- 学到:`bg-xxx` = 背景色;class 名就是样式。

**1-2 🙋 改大小和间距**
- 做:把那个按钮的 `h-8`(高度)改成 `h-10`,`px-2.5`(左右内边距)改成 `px-4`,存盘。
- 会看到:按钮变高变宽。
- 学到:Tailwind 的尺寸是「梯度数字」(2/2.5/4/8...),`h-` 高、`w-` 宽、`p-` 内边距、`m-` 外边距。

**1-3 🙋 加一个鼠标悬停效果**
- 做:在某个按钮 className 里加 `hover:scale-110`,存盘,鼠标移上去看。
- 会看到:悬停时按钮放大。
- 学到:`hover:` 前缀 = 鼠标悬停时才生效;状态前缀是 Tailwind 的核心玩法。

> 做完这三格,Tailwind 你就「会」了——剩下的只是查它有哪些 class(用它官网搜)。

## 阶段 2 · React 的脊梁:组件、状态、属性（动手版）

> React 的全部核心就三样:**组件 = 一个返回界面的函数**;**状态(useState)= 会变的数据,变了界面自动更新**(≈ 你在 Vue 见过的 `ref`/`data`);**属性(props)= 从外面传进来的数据**(≈ Vue 的 props)。

**2-1 🤝 一起读一遍 Toolbar.tsx**
- 做:跟我一起从上到下过一遍这个文件,我指给你看「这是个函数、它 `return` 的是界面、`{...}` 里是 JS」。
- 学到:JSX = 把 HTML 写在 JS 里;一个组件就是一个函数。

**2-2 🙋 加一个会弹窗的按钮**
- 做:照着现有按钮,加一个 `<button onClick={() => alert('hi murphy')}>测试</button>`,存盘,点它。
- 会看到:点击弹出 hi murphy。
- 学到:`onClick={...}` = 点击事件(≈ Vue 的 `@click`);箭头函数 `() => ...` 是「点了之后干什么」。

**2-3 🤝 做一个能切换的按钮(第一次碰 useState)**
- 做:一起加一小段 `const [on, setOn] = useState(false)`,做个按钮点一下在「开/关」之间切。
- 会看到:点击,按钮文字在 开 ↔ 关 之间跳。
- 学到:**这是 React 最关键的一格。** `useState` = 响应式数据,改了它界面自动重画。Redux 你忘了没关系——这个项目根本没用 Redux,`useState` + 我们的引擎 store 就够了。

**2-4 🤝 理解 props:把数据传进组件**
- 做:看 `App.tsx` 怎么把 `engine`、`store` 当属性传给 `<Toolbar engine=... store=... />`。
- 学到:props = 父组件把数据「传下去」给子组件;这就是组件之间怎么通信。

## 阶段 3 · 接上引擎:做一个真功能（这一格通了,你就能做产品了）

> 这是整个编辑器每一个功能的统一套路:**UI 按钮 → 调引擎命令 → 引擎干活**。会了这个,M2~M5 的功能你都能照葫芦画瓢。

**3-1 🤝 加一个「撤销」按钮接到引擎**
- 做:一起在工具栏加个按钮,点击执行 `engine.execCommand('undo')`(命令名我给你)。
- 会看到:改了脑图后点它,撤销上一步。
- 学到:`engine` 就是我们包好的「IMindEngine 适配器」;`execCommand('命令名')` 让引擎干活。React 这边永远只跟 `engine` 这个接口说话,从不直接碰底层库。

**3-2 🤝 让按钮变「智能」(没得撤销时自动灰掉)**
- 做:用 `useCommandState(store, engine, 'undo')` 让按钮在没东西可撤销时自动禁用。
- 会看到:没操作时撤销按钮是灰的,操作后变亮。
- 学到:我们那套「响应式 store」——引擎一变,工具栏自动重新查状态、重画。这就是 `useSyncExternalStore` 在干的事。

> 碰到 TS 类型报错(比如 `: string`、`: number` 那些)就停下喊我,我当场就着那一行给你讲。类型不用提前学,撞上一个学一个。
